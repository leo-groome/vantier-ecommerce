# Vantier Backend — Claude Instructions

## Python Environment

- Uses **pyenv virtualenv**. Virtualenv name: `vantier-backend` (Python 3.12.7)
- `.python-version` file in `backend/` auto-activates it via pyenv-virtualenv
- **Never** use system Python, raw `venv`, or `conda` — always use the pyenv virtualenv
- To install deps: `pip install -e ".[dev]"` from inside `backend/`
- Verify active env: `python --version` should show 3.12.7; `which python` should point to `~/.pyenv/versions/vantier-backend/bin/python`

## Architecture

- **Pattern:** Vertical Slice Architecture (VSA)
- Each feature in `src/features/{name}/` has its own `models.py`, `schemas.py`, `service.py`, `router.py`
- Shared infrastructure only in `src/core/`
- External service wrappers only in `src/integrations/`

## Code Standards

- Python 3.12+, strict type hints everywhere
- Pydantic v2 for all schemas
- SQLAlchemy 2.0 async ORM (`AsyncSession`)
- Google-style docstrings on all public functions
- All business logic in `service.py` — routers are thin

## Database Migrations

- Tool: Alembic with async engine
- Always review autogenerated migrations before committing
- Run `alembic upgrade head` to apply; `alembic downgrade -1` to roll back
- Never autogenerate in CI — migrations are committed artifacts
